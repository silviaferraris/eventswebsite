Specification

  • WebStorm

    WebStorm is a famous IDE for web development that support html, css, javascript.

  • Javascript ES6

    I use javascript es6 that support modules.
    The modules allowed me to simplify the code and reduce redundancies, being able to create a code through the object-oriented paradigm.
    This approach, in addition, it has made it possible to group and reduce API calls into specific files, subdivided by category.

  • JQuery

    To simplify the syntax of client side, I use the framework JQuery that allow to select and edit DOM elements properties in a very intuitive mode.
    The framework is also used to create some animation into the pages.

  • Node.js

    Node.js is a framework that allow to execute javascript code stand-alone.
    I use this framework, together with the express library, to code the backend of the application.
    All the API that I wrote, respect the REST principles.
    The HTML of each page of application is rendered client-side, and this is ensured because the API do not access the html files of the pages, except for sending the html code to the client.

  • Other backend frameworks

    • knex: to establish a connection with the database and execute the queries
    • session: to create the sessions for the logged users
    • passport: to manage the the authentication of the user during the login
    • swaggerUi: to generate the interactive swagger documentation with examples

  • PostgreSQL

    I decided to use PostreSQL because is highly compatible with Heroku (The cloud that host the application).

  • NOTES

    I decided to not use the frameworks Bootstrap and Ajax, because although they could have simplified me a lot of work, I think that in an educational project this instruments are too powerful.
    I wrote by scratch all the html, css and javascript of the client side, having to face challenges (e.g. animation optimization) that I learned to manage.

    I also implemented a dashboard to insert the events, seminars and performers without access directly to the database, simplify and speeding up the insertion of the data. The dashboard and the related API is only accessible by the admin users.
